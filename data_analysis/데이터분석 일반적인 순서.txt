1단계: 문제 정의 (Problem Definition) - "오늘 무슨 요리 할까?"
가장 중요하지만 초보자들이 제일 많이 건너뛰는 단계입니다. 무작정 데이터를 열어보는 것이 아니라, **'내가 이 분석을 통해 무엇을 알고 싶은가?'**를 명확히 하는 과정입니다.

예시: "최근 우리 쇼핑몰의 20대 고객 매출이 왜 떨어졌을까?", "내일 애플 주식은 오를까 내릴까?"

핵심: 목표가 명확해야 어떤 데이터를 수집할지 결정할 수 있습니다.

2단계: 데이터 수집 (Data Collection) - "시장에서 장보기"
문제를 해결하기 위해 필요한 재료(데이터)를 모으는 과정입니다.

방법: * 사내 데이터베이스(SQL)에서 다운로드하기

공공데이터포털, 캐글 등에서 CSV 파일 가져오기 (레벨 1, 2에서 했던 방식)

파이썬 웹 스크래핑(크롤링)이나 API를 통해 실시간 데이터 긁어오기

핵심: 신선하고 좋은 재료를 구하는 것이 맛있는 요리의 기본입니다.

3단계: 데이터 전처리 (Data Preprocessing) - "재료 씻고 다듬기"
데이터 분석가의 업무 시간 중 70~80%를 차지하는 가장 고된 시간입니다. 현실의 데이터는 흙도 묻어있고 상한 부분도 있기 때문입니다.

주요 작업:

결측치 처리: 타이타닉에서 빈칸(NaN)을 평균 나이로 채운(fillna()) 작업입니다.

이상치 제거: 나이가 200살로 잘못 적힌 데이터 등을 찾아내어 수정하거나 삭제합니다.

타입 변환: 텍스트를 날짜 시간 데이터로 바꾼(to_datetime()) 작업입니다.

4단계: 탐색적 데이터 분석 (EDA) - "재료 간 보기"
본격적인 요리에 앞서 재료의 특성을 파악하는 단계입니다. 데이터를 이리저리 쪼개보고, 평균도 내보고, 간단한 그래프도 그려보며 데이터 속에 숨겨진 힌트를 찾습니다.

주요 작업:  groupby()로 그룹별 통계 내기, 상관관계 파악하기, 간단한 막대형/꺾은선 그래프 그려보기 등입니다.

핵심: 이 과정에서 "어? 출퇴근 시간에 따릉이 대여량이 확 튀네?" 같은 중요한 '인사이트(통찰)'를 얻게 됩니다.

5단계: 모델링 및 심층 분석 (Modeling / Analysis) - "본격적인 요리하기"
단순한 통계를 넘어, 기계학습(Machine Learning)이나 통계적 기법을 사용해 미래를 예측하거나 데이터를 분류하는 단계입니다.

주요 작업:

과거 데이터를 바탕으로 내일의 주식 가격 예측하기

비슷한 구매 패턴을 가진 고객들끼리 그룹으로 묶기 (군집화)

비고: 파이썬 기본기를 다진 후, 앞으로 딥러닝이나 머신러닝을 본격적으로 다루시게 되면 이 단계에 가장 많은 공을 들이게 됩니다.

6단계: 결과 시각화 및 커뮤니케이션 (Communication) - "예쁘게 플레이팅해서 서빙하기"
아무리 대단한 분석을 했어도, 다른 사람(팀원, 상사, 고객)이 이해하지 못하면 의미가 없습니다. 분석 결과를 가장 직관적이고 설득력 있게 보여주는 단계입니다.

주요 작업: 맷플롯립(Matplotlib), 씨본(Seaborn) 등을 활용해 목적에 맞는 화려하고 명확한 그래프 그리기, 보고서(대시보드) 작성하기.

데이터 분석은 이 1~6단계를 한 번에 끝내는 것이 아니라, 분석하다가 데이터가 부족하면 다시 2단계로 돌아가고, 결과가 이상하면 3단계로 돌아가는 무한 반복(Iteration)의 과정입니다.






1. 데이터 수집 (Data Collection) : 외부의 데이터를 내 파이썬으로
단순히 엑셀 파일을 더블클릭해서 여는 것이 아니라, 방대한 데이터를 코드로 끌어오는 작업입니다.

정형 데이터 불러오기: 가장 기본이 되는 작업입니다. 주로 pd.read_csv(), pd.read_excel()을 사용하여 로컬 PC나 서버에 있는 파일을 판다스 데이터프레임(표 형태)으로 불러옵니다.

API를 통한 실시간 데이터 수집: 주식 데이터나 날씨 데이터처럼 실시간으로 변하는 정보는 제공 기관의 API를 통해 끌어옵니다. (예: yfinance 라이브러리로 특정 종목의 10년 치 주가와 거래량을 단 한 줄의 코드로 수집)

웹 크롤링(웹 스크래핑): 웹페이지에 흩어져 있는 텍스트나 표 데이터를 파이썬 코드로 긁어모아 데이터프레임으로 변환합니다.

2. 데이터 전처리 (Data Preprocessing) : 분석가의 진짜 실력이 판가름 나는 곳
수집된 데이터는 결코 깨끗하지 않습니다. 이 지저분한 데이터를 AI 모델이나 통계 기법에 넣을 수 있도록 '정제'하는 고된 작업이며, 판다스와 넘파이가 가장 활약하는 무대입니다.

결측치(Missing Value) 처리: 비어있는 칸(NaN)을 어떻게 할지 결정합니다.

찾기: df.isnull().sum()

버리기: 데이터가 충분하다면 빈칸이 있는 행을 삭제합니다. (df.dropna())

채우기: 주식 데이터에서 주말이라 거래가 없어 빈칸이 생겼다면, 금요일 가격으로 채우거나 앞뒤 날짜의 평균을 계산해 넣습니다. (df.fillna())

이상치(Outlier) 제거: 상식에서 벗어난 데이터를 찾아냅니다. 예를 들어 나이가 '999세'로 입력된 데이터를 찾아내어 필터링합니다. (df[df['나이'] < 120])

데이터 타입 변환: 컴퓨터가 계산할 수 있는 형태로 바꿉니다. 예를 들어 '2025년 12월 14일'이라는 문자를 진짜 시간 데이터로 변환(pd.to_datetime())해야 월별, 일별 추세를 분석할 수 있습니다.

파생 변수 생성: 기존 데이터를 조합해 새로운 통찰력을 줄 수 있는 데이터를 만듭니다. (예: '키'와 '몸무게' 데이터를 나누고 곱해 'BMI 지수'라는 새로운 열(Column) 만들기)

3. 탐색적 데이터 분석 (EDA, Exploratory Data Analysis) : 데이터와 대화하기
전처리가 끝난 깨끗한 데이터를 이리저리 쪼개보고 뒤집어보며 숨겨진 패턴과 인사이트를 발굴하는 과정입니다.

요약 통계 확인: 데이터의 전반적인 특징을 한눈에 파악합니다. df.describe()를 치면 평균, 표준편차, 최대/최소값, 상위 25% 값 등이 쫙 펼쳐집니다.

그룹화 및 집계 (Grouping & Aggregation): 특정 기준에 따라 데이터를 묶어봅니다. df.groupby()나 엑셀의 피벗테이블과 같은 pd.pivot_table()을 사용합니다.

상관관계 파악: 두 변수가 얼마나 같이 움직이는지 확인합니다. df.corr()를 통해 금리가 오를 때 주가가 떨어지는지, 광고비를 늘리면 매출이 오르는지 숫자로 확인합니다.

시각화를 통한 패턴 발견: 맷플롯립을 활용해 숫자를 그림으로 바꿉니다.

분포를 볼 때: 히스토그램 (plt.hist())

관계를 볼 때: 산점도 (plt.scatter())

추세를 볼 때: 꺾은선 그래프 (plt.plot())

4. 머신러닝 및 AI 모델링 준비 (Advanced)
이후 본격적인 딥러닝이나 머신러닝(컴퓨터 비전, 시계열 예측 등)으로 넘어가게 된다면, 위에서 판다스와 넘파이로 정성껏 다듬은 데이터가 **AI 모델을 학습시키기 위한 가장 훌륭한 식량(Training Data)**이 됩니다. 전처리를 얼마나 잘했느냐에 따라 AI 모델의 성능이 하늘과 땅 차이로 벌어지게 됩니다.
